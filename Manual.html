Getting Started

Section 1 of 34 · القسم 1 من 34

مرحباً بك في نظام RS · Pearl لإدارة الغسيل

هذا النظام هو تطبيق ويب متكامل لإدارة عمليات الغسيل الفندقي — يتتبع الكميات والإيرادات والأوزان عبر جميع الأقسام يومياً.

ما يمكنك فعله بالنظام:
  • إدخال كميات الغسيل اليومية لكل قسم (غرف، مطعم، سبا، زي موحد، أخرى، تنظيف جاف)
  • مشاهدة الإيرادات والأوزان تُحسب تلقائياً بناءً على الأسعار
  • تتبع الأداء عبر لوحة التحكم والتقارير الشهرية
  • مزامنة البيانات فوراً مع Firebase للوصول من أي جهاز
  • إنشاء تقارير PDF و Excel و CSV للإدارة المالية
  • التنبؤ بالإيرادات والتخطيط للعام القادم

كيفية الوصول:
  1. افتح الرابط في أي متصفح (Chrome أو Safari أو Edge)
  2. أدخل اسم المستخدم وكلمة المرور
  3. انقر على ▶ دخول النظام
  4. ستظهر لوحة التحكم فوراً مع بيانات الشهر الحالي
💡 نصيحة: احفظ الرابط كإشارة مرجعية في متصفحك للوصول السريع. يعمل النظام على الكمبيوتر والجوال والتابلت.
E English
How to open the system:
  • Open your GitHub Pages URL in any browser: https://redasalah1980-pixel.github.io/pearl-laundry/index.html
  • Works on any device — laptop, desktop, tablet, phone. No installation needed.
  • Bookmark the URL on all your devices for one-click access.

🎨 Login Screen — Immersive Dark Design:
  • Full-screen animated background — deep dark space scene with drifting gold particles, nebula glows and occasional shooting stars.
  • A glass card floats in the center — semi-transparent with gold shimmer line at top.
  • RS Pearl monogram with pulsing gold rings at the top of the card.
  • Pearl Management System title in elegant serif font.
  • Underline-style input fields with gold focus glow.
  • Gold gradient ▶ ENTER SYSTEM button.
  • Security badges at the bottom: 🔒 Encrypted · ☁️ Cloud Sync · 🛡️ Licensed.

🔑 Licence Protection:
  • The system is domain-locked — only runs on approved URLs.
  • On your approved domain (github.io) → goes straight to login. No key needed.
  • On any unknown domain → shows a locked screen. A valid licence key is required.
  • To give someone else access → generate a key in ⚙️ Settings → 🔑 Licences.
  • You can revoke any key anytime.

🔐 Login credentials:
  • Default Admin credentials are set during system setup. Contact your system administrator if you need login details.
  • Press Enter or click ▶ ENTER SYSTEM.
  • Team members use their own credentials set in ⚙️ Settings → Team Account.
  • Wrong password → card shakes with red error message.
  • Caps Lock on → amber warning appears automatically.

Top navigation bar after login:
  • Year selector — switch between 2025–2035. Each year is completely independent.
  • ● 1 ONLINE — admin only. Shows how many users are active. Click to see names.
  • ● LIVE = connected to Firebase cloud.  ● OFFLINE = no internet — saves locally, auto-syncs when back online.
  • ⚙️ Your Name — opens Settings (admin only).
  • ⏏ Logout — always visible top right.
💡 Tip: Bookmark the URL on your laptop AND work desktop for instant one-click access. Both go straight to login — no key needed.

Daily Entry — Entering Quantities & Mobile Use
الإدخال اليومي — إدخال الكميات والاستخدام على الجوال
Section 2 of 34 · القسم 2 من 34
ع العربية
أين تجده: انقر على تبويب ✏️ الإدخال في شريط التنقل العلوي.

ما يفعله: إدخال كميات الغسيل اليومية لكل قسم وكل صنف.

طريقة الاستخدام:
  1. اختر الشهر من القائمة المنسدلة في أعلى الصفحة
  2. اختر القسم (غرف / مطعم / سبا / زي موحد / أخرى / تنظيف جاف)
  3. انقر على اليوم المطلوب في التقويم — يفتح جدول الإدخال
  4. أدخل الكميات لكل صنف في العمود المقابل لذلك اليوم
  5. انقر على 💾 حفظ اليوم — يُحفظ فوراً محلياً وعلى السحابة

مؤشرات الحالة في التقويم:
  • رقم أخضر — يوم به بيانات مُدخلة
  • رقم رمادي — لا توجد بيانات لهذا اليوم
  • حدود ذهبية — اليوم المحدد حالياً
💡 نصيحة: أدخل البيانات يومياً للحصول على تقارير دقيقة. يمكن الإدخال من الجوال بنفس الطريقة.
⚠️ مهم: انقر دائماً على زر الحفظ بعد الإدخال. لا يحفظ النظام تلقائياً أثناء الكتابة.
E English
How to enter data:
  1. Select the Month, Day, and Department from the dropdowns.
  2. Type the piece count in the QTY column for each item. Revenue and KG update instantly.
  3. Use the department tabs below the toolbar to switch between departments quickly without changing the day.
  4. Click 💾 Save Day to save. Do this for each department.
The 4 live stat cards (update as you type):
  • Day Total (QR) — total revenue for all departments on the selected day.
  • Day KG — total weight for all departments.
  • Department Revenue — revenue for the currently selected department only.
  • Department Pieces — total piece count for the current department.
Toolbar buttons explained:
  • 💾 Save Day — commits all quantities for current department and day to cloud.
  • 📋 Copy Day — copies all quantities to clipboard. A blue badge appears to confirm.
  • 📌 Paste Day — grayed out until you copy. Switch to any day or department then click — fills instantly. Amber ✕ clears the clipboard.
  • 📝 Receiving Log — opens the batch drawer on the right. Red badge = pending batches.
  • 📋 Bulk Paste — paste multiple days from Excel at once.
  • 📂 Import Excel — import a full .xlsx file.
  • 🗑 Clear ▾ — clear data for a day, date range, or entire month.
Copy & Paste workflow:
  1. Select department + day → click 📋 Copy Day.
  2. Switch to a different day or department.
  3. Click 📌 Paste Day — quantities fill automatically.
  4. Click 💾 Save Day. The clipboard stays active so you can keep pasting to more days.
  5. Click the amber ✕ button when done to clear the clipboard.
⚠️ Important: Always click 💾 Save Day after entering or pasting. Navigating away without saving will lose that day's data.

🔒 Locked Prices — Permanent Price Protection:

Every time you click 💾 Save Day, the system permanently locks the price of each item at that exact moment. This means:
  • If you change prices next month, all previously saved data keeps its original prices forever.
  • Finance calculations, Dashboard revenue, and all reports always use the price that was active when the data was saved.
  • No schedules or manual steps needed — protection is automatic on every save.
⚠️ Price Conflict Warning:

If you try to save a day that was previously saved with different prices, the system shows a warning:
  • 🔒 Keep Old Prices — saves the quantity correction but keeps the original locked prices. Use this when fixing a typo.
  • 🔄 Update Prices — saves with current prices. Use this if you intentionally want to reprice.
  • Cancel — nothing saved.
💡 Tip: When correcting a quantity mistake on an old day, always choose Keep Old Prices to protect the revenue figures that were already reported.

📱 Mobile Use:

Open the same link on your phone or tablet — the system detects the screen size and adapts automatically. On mobile you will see:
  • A bottom navigation bar with Dashboard, Entry, Finance, Benchmark, and a ⋯ More button for other tabs.
  • Touch-friendly larger buttons and stat cards.
  • The entry table scrolls horizontally so all columns are accessible.
  • All data syncs instantly — same Firebase database as desktop.
💡 Tip: Bookmark the link on your phone home screen for one-tap access. Works on iOS Safari and Android Chrome.

Receiving Log — Multiple Batches Per Day
سجل الاستلام — دفعات متعددة في اليوم
Section 3 of 34 · القسم 3 من 34
ع العربية
أين تجده: تبويب ✏️ الإدخال ← زر 📝 سجل الاستلام.

ما يفعله: تسجيل دفعات استلام متعددة في نفس اليوم — مفيد عندما يصل الغسيل على دفعات خلال اليوم.

طريقة الاستخدام:
  1. انقر على 📝 سجل الاستلام
  2. اختر اليوم والقسم
  3. أضف دفعة بالنقر على + إضافة دفعة
  4. أدخل الكميات لكل دفعة
  5. انقر ✅ تطبيق على اليوم — تُجمع كل الدفعات وتُحفظ كإجمالي اليوم
💡 نصيحة: استخدم سجل الاستلام عندما يصل غسيل الصباح والمساء بشكل منفصل.
E English
When to use: When laundry arrives in separate deliveries during the day. Record each batch separately and the system adds them together.
  1. Click 📝 Receiving Log — the batch drawer slides in from the right.
  2. Select the Day and Department.
  3. Click + Add Batch for the first delivery and enter quantities.
  4. Click + Add Batch again for each additional delivery.
  5. When all batches are entered, click ✅ Apply to Day — all batches are summed and the entry table fills automatically.
  6. Click 💾 Save Day to commit.
  • 📋 Copy button duplicates a batch. 🗑 deletes a single batch.
  • Batches are saved in the browser — closing and reopening the drawer keeps them.
  • The red badge on the button shows how many batches are pending.
💡 Tip: Use Receiving Log when you have morning and evening deliveries. It prevents manual addition errors.

Bulk Paste — Paste Multiple Days From Excel
اللصق الجماعي — لصق أيام متعددة من Excel
Section 4 of 34 · القسم 4 من 34
ع العربية
أين تجده: تبويب ✏️ الإدخال ← زر 📋 لصق جماعي.

ما يفعله: لصق بيانات عدة أيام دفعة واحدة من Excel — يوفر وقت الإدخال اليدوي.

طريقة الاستخدام:
  1. انقر على ⬇️ تحميل القالب للحصول على ملف Excel بالتنسيق الصحيح
  2. أدخل بياناتك في القالب
  3. حدد الخلايا في Excel وانسخها (Ctrl+C)
  4. انقر داخل منطقة اللصق في النظام
  5. الصق (Ctrl+V) — يعرض النظام معاينة البيانات
  6. راجع البيانات وانقر ✅ تطبيق
⚠️ مهم: تأكد من تطابق ترتيب الأصناف في Excel مع ترتيب النظام. استخدم القالب المُضمن دائماً.
E English
When to use: Entering data for a past month from an existing Excel file. Saves hours of manual entry.
  1. Click 📋 Bulk Paste in the toolbar.
  2. Set Year, Month, Day From, Day To.
  3. Select the exact Department — do not leave it on "All Departments".
  4. In Excel: select the item names column + day columns only. No headers, no totals row.
  5. Press Ctrl+C in Excel → click inside the paste box in the system → press Ctrl+V.
  6. Check the preview table — confirm items are matched correctly.
  7. Click Apply All Rows.
⚠️ Important: Always select a specific department (e.g. Rooms Linen) before pasting. The system maps Row 1 → Item 1, Row 2 → Item 2 in order. If you select "All Departments" it uses name matching which can be unreliable for similar names like King/Queen/Full sheets.
  • Blank cells = 0. You can paste a full month (Day 1–31) in one action.
  • Paste one department at a time. Six departments = six separate pastes.
  • After applying, the system jumps to the pasted year/month so you can verify immediately.
💡 Tip: Check the item order in ✏️ Daily Entry first to confirm it matches your Excel row order before pasting.

Import Excel — Full .xlsx File Import
استيراد Excel — استيراد ملف .xlsx كامل
Section 5 of 34 · القسم 5 من 34
ع العربية
أين تجده: تبويب ✏️ الإدخال ← زر 📂 استيراد Excel.

ما يفعله: استيراد ملف .xlsx كامل يحتوي بيانات يوم واحد لقسم واحد.

طريقة الاستخدام:
  1. انقر على ⬇️ تحميل القالب لتنزيل القالب الخاص بالقسم المطلوب
  2. أدخل الكميات في القالب واحفظه
  3. انقر على 📂 استيراد Excel واختر الملف
  4. يعرض النظام معاينة — راجع البيانات
  5. انقر ✅ استيراد لتطبيق البيانات
💡 نصيحة: استخدم الاستيراد لنقل بيانات من أنظمة أخرى أو عندما تكون الكميات كبيرة ومعقدة.
E English
When to use: You have a complete .xlsx file with one sheet per department. A ready-to-use example template is available for download from the system.
  1. Click 📂 Import Excel in the toolbar.
  2. Set Year, Month, Day From, Day To.
  3. Choose All Sheets or a specific department.
  4. Click Choose File and select your .xlsx file.
  5. Review the preview table — confirm item names on the left match system rows on the right.
  6. Click Import All Data.
Excel file structure — exact rules:
  • Sheet names must match exactly: Rooms Linen · F & B · Spa & Pool · Uniform · Others · Dry Cleaning. Any other name is ignored.
  • Rooms Linen: Col A = #, Col B = Item Name, Col C = Day 1, Col D = Day 2 ... Col AG = Day 31.
  • All other sheets: Col A = #, Col B = Item Name, Col C = Price, Col D = Weight, Col E = Day 1, Col F = Day 2 ... Col AJ = Day 31.
  • Row layout (Rooms Linen): Row 1 = title, Row 2 = column headers, Row 3+ = item data.
  • Row layout (F&B, Uniform, Others, Dry Cleaning): Row 1 = title, Row 2 = headers, Row 3+ = item data.
  • Row layout (Spa & Pool): Row 1 = title, Row 2 = headers, Row 3 = sub-header, Row 4+ = item data.
⚠️ Important: Items are matched by position only — Row 1 = Item 1, Row 2 = Item 2. Do NOT add, remove, or reorder rows. Do NOT add blank rows between items. Leave rows with no data as zeros.
Month length handling:
  • Set Day To to match the month: 28 for February, 30 for April/June/Sept/Nov, 31 for other months.
  • The system only reads the columns you specify. Extra day columns beyond Day To are ignored.
  • For February: fill days 1–28 and leave day columns 29–31 blank.
💡 Tip: Use the example template available from the system — it has all sheets pre-built with correct structure, item names in the exact right order, and instructions inside.

استيراد شهر كامل — جميع الأقسام دفعة واحدة

أين تجده: تبويب ✏️ الإدخال ← زر 📅 استيراد شهر كامل.

ما يفعله: استيراد ملف Excel واحد يحتوي بيانات شهر كامل لجميع الأقسام دفعة واحدة.

طريقة الاستخدام:
  1. انقر على ⬇️ تحميل قالب الشهر الكامل
  2. أدخل بيانات جميع الأيام لجميع الأقسام في القالب
  3. احفظ الملف وارفعه عبر 📂 رفع الملف
  4. يعرض النظام ملخص الاستيراد — راجعه
  5. انقر ✅ تطبيق لاستيراد الشهر كاملاً
⚠️ مهم: الاستيراد يستبدل بيانات الأيام الموجودة. تأكد من عمل نسخة احتياطية أولاً.
E English
What it does: Upload one Excel file containing all departments and all days for a full month — in one click.

How to use:
  1. Go to Entry tab → click 📅 Import Full Month.
  2. Select the Year and Month you want to import.
  3. Click ⬇️ Download Template — saves an Excel file with one sheet per department.
  4. Open the Excel file and fill in quantities for each day and item.
  5. Come back → choose the filled file → system shows a preview of what will be imported.
  6. Click ✅ Import All Departments — all data saved at once.

Excel file structure:
  • One sheet per department (sheet names must match department names).
  • First row = headers: Day | Item 1 | Item 2 | Item 3 | ...
  • Each row after = one day: 1, 22, 6, 18, ...
  • Leave cells blank to skip — existing data is kept.
  • Use the downloaded template to get the correct format automatically.
⚠️ Important: Blank cells are skipped — existing data for that item/day stays. Only cells with numbers are imported. Always download the template first to get correct item names and column order.
💡 Tip: Download the template at start of month, fill it throughout the month as a daily log, then import at end of month in one shot.

December 31 Carry-In — January Finance Accuracy
ترحيل 31 ديسمبر — دقة الحسابات المالية لشهر يناير
Section 7 of 34 · القسم 7 من 34
ع العربية
ما هو ترحيل 31 ديسمبر؟

في بعض الفنادق، الغسيل الذي يُستلم في 31 ديسمبر لا يُسلَّم إلا في الأول من يناير. هذه الميزة تضمن أن الإيرادات تُسجَّل في الشهر الصحيح.

أين تجده: تبويب ✏️ الإدخال ← اختر ديسمبر ← اليوم 31 ← قسم ← زر 📥 ترحيل يناير.

طريقة الاستخدام:
  1. أدخل كميات 31 ديسمبر كالمعتاد
  2. للكميات التي ستُرحَّل ليناير، انقر على 📥 ترحيل يناير
  3. أدخل الكميات المُرحَّلة
  4. احفظ — تُضاف تلقائياً إلى اليوم الأول من يناير
💡 نصيحة: استخدم هذه الميزة إذا كانت سياسة الفندق تعتمد على تاريخ التسليم وليس الاستلام.
E English
Why this exists: Items received on December 31 are posted to finance on January 1 of the next year. Year 2025 exists only to hold this carry-in data.
  1. Switch the Year selector to 2025.
  2. Only December 2025 is available.
  3. Go to ✏️ Daily Entry → select Day 31.
  4. Enter quantities per department.
  5. Click 💾 Save Day for each department.
Effect: Finance Posting → January 2026 → the amber row on Day 1 automatically reads this data. Enter it once and it is always correct.
💡 Tip: If you skip this, the January Finance Posting amber row will show zero instead of the correct carry-in amount.

What it is: A separate standalone HTML file (emergency_restore.html) that can recover your data directly from Firebase using the REST API — completely independent of the main system.

🚨 Use this tool ONLY when: the main system will not load, you are on a new device with no data, or the normal restore functions inside the system are not working.
Key advantage over the built-in restore:
  • Works without logging in — open the file directly in any browser
  • Does not need the main system to be working at all
  • Uses Firebase REST API directly — bypasses the Firebase SDK completely
  • Can download your raw data as JSON even if nothing else works

How to use — 3 steps:
  1. Open emergency_restore.html in any browser (double-click the file)
  2. Click 🔌 Test Connection — confirms Firebase is reachable. If it fails, check your internet or Firebase rules.
  3. Choose your restore method:

Available actions:
  • 🔍 Check All Data — shows a summary of everything in Firebase
  • 📊 Check Entry Data — shows entry data per year with entry counts
  • 🏨 Check Occupancy — shows occupancy data per month
  • 💾 Check Backup Versions — lists all saved backup versions
  • ✅ Restore ALL Entry Data — pulls all entry data from Firebase into this browser's localStorage
  • 🏨 Restore Occupancy — restores occupancy for all years and months
  • ☁️ Restore from Latest Backup — uses the latest cloud backup snapshot instead of live data
  • ⬇️ Download ALL Firebase Data — downloads the complete raw Firebase database as JSON

After restoring:
  1. The tool writes data directly to localStorage in your browser
  2. Press Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows) to hard-refresh
  3. Open your main RS Pearl system — data should appear immediately
  4. If it does not appear, use the 🔃 Force Re-Sync button in Settings → Diagnostics

Firebase Rules note:
If the connection test fails with a permission error, your Firebase rules may require authentication. Temporarily set rules to public read, restore your data, then change them back:
{"rules":{".read":true,".write":true}}
⚠️ Important: Change Firebase rules back to authenticated access after you finish the emergency restore. Open rules: Firebase Console → Realtime Database → Rules → Publish.
💡 Tip: Keep the emergency_restore.html file saved separately from the main system — on your desktop, USB drive, or email it to yourself. If the main system ever breaks, you want this file available even without internet access to the main system.
